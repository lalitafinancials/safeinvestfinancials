🚀 Go Live — Baby Steps

Follow these steps in order. Click each phase to expand it. Total time: about 30 minutes.

1
Create GitHub Account
~2 min
1
Go to github.com → click Sign up
2
Choose a username — e.g. johnfinancials
3
Enter your email and create a password
4
Verify your email address (check inbox for code)
2
Create Repository
~1 min
1
Once logged in → click the green "+ New" button (top left)
2
Repository name: safeinvest-john
3
Select Public
4
Click Create repository
3
Upload Your Files
~3 min
1
Make sure your generated file is named exactly: index.html
2
Make sure your photo is named: photo.jpg
3
Copy logo.png from Lalita's repo (or use your own)
4
In your GitHub repo → click "Add file" → "Upload files"
5
Drag and drop all 3 files: index.html, your photo, logo.png
6
Click Commit changes → Commit changes again
4
Enable GitHub Pages
~2 min
1
In your repo → click Settings tab (gear icon, far right)
2
In the left sidebar → scroll down → click Pages
3
Under Branch → select main → select / (root)
4
Click Save → wait 3 minutes
5
Your site goes live at: https://johnfinancials.github.io/safeinvest-john
6
Open that URL in your browser to verify it works ✅
5
Set Up Google Sheet + Contact Logging
~10 min
1
Go to sheets.google.com → click + to create new sheet
2
Name it: John Contact Log
3
Right-click the Sheet1 tab at bottom → Rename → type Contact Log
4
Copy the Sheet ID from the URL (the long string between /d/ and /edit)
5
In the sheet → click Extensions → Apps Script
6
Delete existing code (Ctrl+A → Delete) → paste the SafeInvest Contact Handler script
7
Update: SPREADSHEET_ID = your Sheet ID, EMAIL_1 = your email
8
Click 💾 Save → select testSheet from dropdown → click ▶ Run
9
If permission popup → click Advanced → Go to (unsafe) → Allow
10
Check your Google Sheet — you should see a test row ✅
11
Select testEmail → Run → check your email inbox ✅
12
Click Deploy → New deployment → ⚙️ gear → Web app
13
Execute as: Me · Who has access: Anyone → click Deploy
14
Copy the Web App URL → update SCRIPT_URL in index.html → re-upload to GitHub
6
Custom Domain (Optional)
~15 min
1
In GitHub → Settings → Pages → Custom domain → enter: your domain
2
In your DNS provider (Wix/Namecheap/GoDaddy) → DNS Records
3
Delete existing A records → Add 4 new A records (Host = @):
185.199.108.153 · 185.199.109.153 · 185.199.110.153 · 185.199.111.153
4
Add CNAME record: Host = www, Value = username.github.io
5
Wait 30–60 minutes → return to GitHub Pages → check the Enforce HTTPS box
6
Visit your domain — 🔒 padlock should appear ✅
